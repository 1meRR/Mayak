# Decentra Call Messenger

MVP-монорепа для децентрализированного мессенджера с упором на видеозвонки:

- **Flutter** клиент для Android / iOS / Windows / macOS / Linux
- **Rust** signaling server
- **WebRTC mesh** для P2P-видеозвонков 1:1 и маленьких групп
- встроенный текстовый чат внутри комнаты
- адаптивный тёмный UI для desktop + mobile

## Что здесь уже работает

- создание и вход в комнату
- локальное аудио/видео
- обмен offer / answer / ICE через WebSocket signaling
- P2P-подключение между участниками через WebRTC
- сетка участников
- mute / camera on-off / switch camera
- встроенный чат комнаты
- сохранение имени, URL сервера и недавних комнат

## Что важно понимать честно

Это **рабочий MVP-каркас**, а не finished production-мессенджер уровня Telegram/Signal/Zoom.

В текущем виде:

- signaling сервер централизован
- звонки идут по **mesh P2P**
- для части NAT-сценариев нужен **TURN**
- end-to-end crypto для сообщений поверх отдельного протокола ещё не добавлен
- офлайн-репликация и полноценный decentralised identity слой ещё не добавлены

Для маленьких комнат (2–4 человека) такой подход хорош. Для больших групп потом нужен SFU или гибридная архитектура.

## Быстрый старт

### 1. Поднять signaling server

Установи Rust, затем:

```bash
cd apps/signaling-server
cargo run
```

По умолчанию сервер слушает:

```text
0.0.0.0:8080
```

Health-check:

```text
GET http://127.0.0.1:8080/health
```

WebSocket endpoint:

```text
ws://127.0.0.1:8080/ws
```

---

### 2. Подготовить Flutter-клиент

Так как в архиве лежит уже написанный код приложения, но не лежат autogenerated platform runner файлы Flutter SDK, сделай так:

```bash
cd apps/client
flutter create . --platforms=android,ios,windows,macos,linux
flutter pub get
```

Эта команда создаст недостающие platform-папки и не сотрёт твой `lib/` и `pubspec.yaml`.

Потом:

```bash
flutter run -d windows
```

или:

```bash
flutter run -d android
flutter run -d ios
flutter run -d macos
flutter run -d linux
```

## Адрес signaling server для разных сценариев

### Windows / macOS / Linux desktop
```text
ws://127.0.0.1:8080/ws
```

### Android emulator
```text
ws://10.0.2.2:8080/ws
```

### iOS simulator
```text
ws://127.0.0.1:8080/ws
```

### Реальный телефон в одной Wi‑Fi сети с твоим ПК
```text
ws://<LAN-IP-ТВОЕГО-КОМПЬЮТЕРА>:8080/ws
```

Например:

```text
ws://192.168.0.15:8080/ws
```

## TURN / NAT

Для production тебе почти наверняка нужен TURN (например, coturn), иначе часть пользователей за symmetric NAT не смогут соединиться напрямую.

Сейчас в проекте зашиты только публичные STUN-серверы для простого локального теста.

4. Добавить E2E encryption для сообщений и метаданных
5. Сделать local-first message store
6. Добавить push / call invites / background resume
7. Перейти с mesh на SFU или hybrid topology для групповых звонков
